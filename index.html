About

Q is a simple stand-alone query abstraction layer.

It is not a database abstraction layer.

Q tries to eliminate redundancy of writing SQL across different database platforms.

Q is simple. : )

FAQ

Why call it Q?

It fits with Q::uery, which is the singleton constructor used to create a new query.

Why use Q?

If you don't want to, don't.

Q was created to eliminate writing SQLs over and over for each different database engine. It hopes to make things easier and more portable by providing a common interface to write SQL queries.

Why not use projects like Doctrine which already provides such a feature with a hundred more?

Short answer: Why not? Sure!

Long answer: Usually when developing simple applications I do not require a full-fledged ORM like Doctrine. However, I still need to make it portable and writing conditions for each database engine everytime isn't very convinient. With Q, I can easily port the queries across platforms.

What are the features of Q?

Query abstraction, that's it! (Although if you want, you can plug other things in, like a database abstraction layer but that's up to you.)
